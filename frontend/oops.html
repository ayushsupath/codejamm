1. Classes & Objects

A Class is a blueprint or template (like a car design). An Object is the actual physical entity created from that blueprint (like a red Toyota Corolla).
Car Class Blueprint
Attributes: color, speed
Methods: drive(), brake()
Red Car Object
Blue Car Object
Java 
class Car {
    String color;
    int maxSpeed;
    
    // Constructor
    Car(String c, int s) {
        color = c;
        maxSpeed = s;
    }
    
    void displayDetails() {
        System.out.println("Color: " + color + ", Max Speed: " + maxSpeed);
    }
}

public class Main {
    public static void main(String[] args) {
        // Creating objects from the Car blueprint
        Car redCar = new Car("Red", 200);
        Car blueCar = new Car("Blue", 180);
        
        redCar.displayDetails();
        blueCar.displayDetails();
    }
}
Q: If 'Animal' is a class, what is 'Dog'?
A variable
An object (instance of a class)
A method
Correct! 'Dog' is the concrete manifestation/object of the abstract 'Animal' blueprint.

2. Encapsulation

Wrapping data (variables) and code acting on the data (methods) together as a single unit. It hides the internal state of an object from the outside world (Data Hiding).
Java 
class BankAccount {
    // Private: Cannot be accessed directly from outside
    private double balance;
    
    public BankAccount(double initialBalance) {
        if(initialBalance > 0) balance = initialBalance;
    }
    
    // Public getter
    public double getBalance() {
        return balance;
    }
    
    // Public setter with validation
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println("Deposited: " + amount);
        }
    }
}

public class Main {
    public static void main(String[] args) {
        BankAccount myAccount = new BankAccount(500);
        // myAccount.balance = 1000; // ERROR! private access
        
        myAccount.deposit(200);
        System.out.println("Current Balance: " + myAccount.getBalance());
    }
}
Q: What is the primary purpose of getter and setter methods?
To make code run faster
To control access and validate data modifications
To convert classes to interfaces
Correct! They protect the internal state.
