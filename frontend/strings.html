1. String Immutability

In Java, objects of type String are immutable. When you modify a String, you are actually creating a brand new String object in the String Pool!

Why Immutable?

  • Security: Parameters like DB URLs are passed as Strings.
  • Synchronization: Inherently thread-safe.
  • Caching: The String Constant Pool optimizes memory.
Java 
public class StringImmutability {
    public static void main(String[] args) {
        String s1 = "Code";
        String s2 = "Code";       // Points to the same object in pool
        
        System.out.println(s1 == s2); // true
        
        s1 = s1 + "Jamm";         // s1 now points to a new object "CodeJamm"
        System.out.println(s2);   // "Code" remains unchanged
    }
}

2. Essential String Methods

Java 
public class StringMethods {
    public static void main(String[] args) {
        String str = "Hello World";
        
        // 1. length()
        System.out.println(str.length());  // 11
        
        // 2. charAt()
        System.out.println(str.charAt(0));  // H
        
        // 3. substring(start, end)
        System.out.println(str.substring(0, 5));  // Hello
        
        // 4. indexOf()
        System.out.println(str.indexOf("o")); // 4 (first 'o')
        
        // 5. toLowerCase() / toUpperCase()
        System.out.println(str.toUpperCase()); // HELLO WORLD
        
        // 6. trim() - removes padded whitespace
        String dirty = "  trim me  ";
        System.out.println(dirty.trim()); // "trim me"
        
        // 7. split()
        String csv = "apple,banana,orange";
        String[] fruits = csv.split(","); // ["apple", "banana", "orange"]
        
        // 8. replace()
        System.out.println(str.replace('o', 'a')); // "Hella Warld"
    }
}

3. StringBuilder & StringBuffer

Since regular Strings are immutable, modifying them in loops creates thousands of dead objects. Use StringBuilder for mutable, fast string building.
  • StringBuilder: Fast, Not Thread Safe (Use in single-threaded ops)
  • StringBuffer: Slower, Thread Safe (Synchronized methods)
Java 
public class BuilderExample {
    public static void main(String[] args) {
        StringBuilder sb = new StringBuilder("Hello");
        
        // Append
        sb.append(" World"); // Modifies the actual object natively
        System.out.println(sb); // Hello World
        
        // Insert
        sb.insert(5, " Beautiful");
        System.out.println(sb); // Hello Beautiful World
        
        // Reverse (very useful for algorithms!)
        sb.reverse();
        System.out.println(sb); // dlroW lufituaeB olleH
    }
}
