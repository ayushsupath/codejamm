Detailed OOPs Concepts - Easy Way

A. Classes and Objects

Simple Definition:
- Class: Blueprint/template (like a building plan)
- Object: Real instance created from class (actual building)
Real-Life Example:
- Class = "Car" blueprint (has properties: color, model, speed)
- Objects = Your actual car, neighbor's car (different colors/models)
Java 
// Class definition - Blueprint
class Car {
    String color;
    String model;
    int speed;
    
    void drive() {
        System.out.println("Car is driving at " + speed + " km/h");
    }
}

// Creating objects - Real instances
public class Main {
    public static void main(String[] args) {
        Car myCar = new Car();  // Object 1
        myCar.color = "Red";
        myCar.model = "Tesla";
        myCar.speed = 120;
        myCar.drive();
        
        Car yourCar = new Car();  // Object 2
        yourCar.color = "Blue";
        yourCar.model = "BMW";
        yourCar.speed = 100;
        yourCar.drive();
    }
}

Key Points:

  • Class is declared once
  • Multiple objects can be created from one class
  • Each object has its own data
  • Objects are created using 'new' keyword

B. Encapsulation

Simple Definition:
- Wrapping data (variables) and code (methods) together
- Hiding internal details from outside world
- Like a capsule that protects medicine inside
Real-Life Example:
- TV remote: You press buttons (public interface)
- Internal circuits are hidden (private data)
- You don't need to know how it works internally
Java 
class BankAccount {
    // Private variables - Hidden from outside
    private double balance;
    private String accountNumber;
    
    // Constructor
    public BankAccount(String accNum, double initialBalance) {
        accountNumber = accNum;
        balance = initialBalance;
    }
    
    // Public method to deposit - Controlled access
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println("Deposited: " + amount);
        }
    }
    
    // Public method to withdraw - Controlled access
    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.println("Withdrawn: " + amount);
        } else {
            System.out.println("Insufficient balance!");
        }
    }
    
    // Public method to check balance - Read-only access
    public double getBalance() {
        return balance;
    }
}

public class Main {
    public static void main(String[] args) {
        BankAccount acc = new BankAccount("12345", 1000);
        
        // Can't access: acc.balance = 50000; // ERROR - private!
        // Must use public methods:
        acc.deposit(500);
        acc.withdraw(200);
        System.out.println("Balance: " + acc.getBalance());
    }
}

Key Points:

  • Use 'private' for variables (data hiding)
  • Use 'public' methods to access data (getters/setters)
  • Protects data from unauthorized access
  • Provides controlled access through methods

C. Inheritance

Simple Definition:
- Child class inherits properties from parent class
- Reusing code from existing class
- Like children inheriting traits from parents
Real-Life Example:
- Parent: "Vehicle" (has wheels, engine, fuel)
- Children: Car, Bike, Truck (inherit vehicle properties + add their own)
Java 
// Parent class (Superclass)
class Animal {
    String name;
    
    void eat() {
        System.out.println(name + " is eating");
    }
    
    void sleep() {
        System.out.println(name + " is sleeping");
    }
}

// Child class 1 (Subclass)
class Dog extends Animal {
    void bark() {
        System.out.println(name + " is barking: Woof! Woof!");
    }
}

// Child class 2 (Subclass)
class Cat extends Animal {
    void meow() {
        System.out.println(name + " is meowing: Meow! Meow!");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.name = "Tommy";
        dog.eat();      // Inherited from Animal
        dog.sleep();    // Inherited from Animal
        dog.bark();     // Own method
        
        Cat cat = new Cat();
        cat.name = "Pussy";
        cat.eat();      // Inherited from Animal
        cat.sleep();    // Inherited from Animal
        cat.meow();     // Own method
    }
}

Key Points:

  • Use 'extends' keyword
  • Child gets all parent's non-private members
  • Child can add its own unique methods
  • Promotes code reusability
  • Java supports single inheritance only (one parent)

D. Polymorphism

Simple Definition:
- "Poly" = Many, "Morph" = Forms
- Same method name, different behaviors
- Like one person playing multiple roles
Types:
1. Method Overloading (Compile-time Polymorphism)
2. Method Overriding (Runtime Polymorphism)

Method Overloading:

Java 
class Calculator {
    // Same method name, different parameters
    
    int add(int a, int b) {
        return a + b;
    }
    
    int add(int a, int b, int c) {
        return a + b + c;
    }
    
    double add(double a, double b) {
        return a + b;
    }
}

public class Main {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        System.out.println(calc.add(5, 10));           // Calls 1st method
        System.out.println(calc.add(5, 10, 15));       // Calls 2nd method
        System.out.println(calc.add(5.5, 10.5));       // Calls 3rd method
    }
}

Method Overriding:

Java 
class Animal {
    void sound() {
        System.out.println("Animal makes sound");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Dog barks: Woof!");
    }
}

class Cat extends Animal {
    @Override
    void sound() {
        System.out.println("Cat meows: Meow!");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a1 = new Dog();
        Animal a2 = new Cat();
        
        a1.sound();  // Output: Dog barks: Woof!
        a2.sound();  // Output: Cat meows: Meow!
    }
}

Key Points:

  • Overloading: Same name, different parameters (compile-time)
  • Overriding: Same signature in parent & child (runtime)
  • Use @Override annotation for clarity

E. Abstraction

Simple Definition:
- Hiding complex implementation details
- Showing only essential features
- Like driving a car without knowing engine internals
Java Abstract Class 
// Abstract class - Cannot create object directly
abstract class Shape {
    String color;
    abstract double calculateArea(); // Abstract method
    
    void displayColor() { System.out.println("Color: " + color); }
}

class Circle extends Shape {
    double radius;
    @Override
    double calculateArea() { return 3.14 * radius * radius; }
}
Java Interface 
// Interface - 100% abstraction
interface Drawable {
    void draw();  // Abstract by default
}

class Circle implements Drawable {
    @Override
    public void draw() { System.out.println("Drawing a circle"); }
}

Key Points:

  • Abstract class: Can have both abstract & concrete methods
  • Interface: All methods are abstract (before Java 8)
  • Abstract class uses 'extends', Interface uses 'implements'
  • Cannot create objects of abstract class/interface

F. Constructor

Simple Definition:
- Special method to initialize objects
- Called automatically when object is created
- Same name as class name
- No return type (not even void)
Java 
class Student {
    String name;
    
    // 1. Default Constructor
    Student() {
        name = "Unknown";
    }
    
    // 2. Parameterized Constructor
    Student(String n) {
        name = n;
    }
    
    // 3. Copy Constructor
    Student(Student s) {
        name = s.name;
    }
}

G. 'this' and 'super' Keywords

'this': Refers to current object, resolves name conflicts.
'super': Refers to parent class object, accesses parent methods/constructors.
Java 
class Parent {
    int x = 10;
    Parent() { System.out.println("Parent constructor"); }
}

class Child extends Parent {
    int x = 20;
    
    Child() {
        super();  // Calls parent constructor
    }
    
    void setValue(int x) {
        this.x = x;  // 'this.x' = instance variable, 'x' = parameter
    }
    
    void display() {
        System.out.println("Parent x: " + super.x);
    }
}

Quick Reference Cheat Sheets

A. Java Basics Cheat Sheet

  • Data Types: int, long, float, double, boolean, char, String
  • Operators: Arithmetic (+, -, *, /), Logical (&&, ||, !), Relational (==, !=, >, <)
  • Control Flow: if-else, switch, for, while, do-while
  • Arrays: int[] arr = new int[5]; arr[0] = 1;

B. Collections Framework

  • List: ArrayList (fast iteration), LinkedList (fast insertions)
  • Set: HashSet (no duplicates), TreeSet (sorted), LinkedHashSet (insertion order)
  • Map: HashMap (key-value), TreeMap (sorted key), LinkedHashMap
  • Queue: PriorityQueue, Deque

C. Exception Handling

  • try-catch-finally: Catching and handling runtime errors smoothly.
  • throw vs throws: 'throw' throws a specific error locally, 'throws' declares what a method can throw.
  • Checked vs Unchecked: Checked (compile time, e.g. IOException), Unchecked (runtime, e.g. NullPointerException).

D. String Methods

  • length(), charAt(), substring()
  • indexOf(), contains(), replace()
  • toLowerCase(), toUpperCase()
  • split(), trim(), equals(), equalsIgnoreCase()
