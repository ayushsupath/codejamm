2. Lists: Ordered Collections

Lists preserve insertion order and allow duplicate elements.

Implementation Best Use Case Avoid When
ArrayList Fast random access via index, iterating Frequent insertions/removals at beginning
LinkedList Frequent insertions/removals at ends Need fast random access to middle elements
Java 
import java.util.*;

public class ListDemo {
    public static void main(String[] args) {
        List<String> arrList = new ArrayList<>();
        arrList.add("Apple");
        arrList.add("Banana");
        arrList.add("Apple"); // Duplicates allowed
        
        System.out.println("ArrayList: " + arrList);
        System.out.println("Element at index 1: " + arrList.get(1)); // Fast O(1)
        
        List<String> linkList = new LinkedList<>(arrList);
        ((LinkedList<String>)linkList).addFirst("Zero"); // Fast insertion O(1)
        System.out.println("LinkedList: " + linkList);
    }
}

3. Sets: Unique Collections

Sets do not allow duplicate elements. Extremely useful for finding unique items.

Java 
import java.util.*;

public class SetDemo {
    public static void main(String[] args) {
        Set<Integer> hashSet = new HashSet<>();
        hashSet.add(10);
        hashSet.add(5);
        hashSet.add(20);
        hashSet.add(10); // Duplicate will be ignored!
        
        // Output order is unpredictable
        System.out.println("HashSet: " + hashSet); 
        System.out.println("Contains 5? " + hashSet.contains(5)); // Extremely fast O(1)
        
        // If sorting is needed, use TreeSet
        Set<Integer> treeSet = new TreeSet<>(hashSet);
        System.out.println("TreeSet (Sorted): " + treeSet); // [5, 10, 20]
    }
}

4. Maps: Key-Value Pairs

Think of a real-world dictionary. Maps map unique Keys to Values (which can be duplicate).
Java 
import java.util.*;

public class MapDemo {
    public static void main(String[] args) {
        Map<String, Integer> scores = new HashMap<>();
        
        // Putting data
        scores.put("Alice", 95);
        scores.put("Bob", 80);
        scores.put("Charlie", 90);
        
        // Overwriting a key
        scores.put("Bob", 85); // Bob's score is updated
        
        System.out.println(scores);
        System.out.println("Alice's score: " + scores.get("Alice"));
        
        // Iterating over Map
        for (Map.Entry<String, Integer> entry : scores.entrySet()) {
            System.out.println(entry.getKey() + " -> " + entry.getValue());
        }
    }
}
