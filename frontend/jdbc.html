1. Introduction

JDBC (Java Database Connectivity) is a standard API that defines how a Java application can access and interact with databases (like MySQL, PostgreSQL, Oracle) to execute SQL statements.

The Core Components:

  1. DriverManager: Manages a list of database drivers. Maps connection requests to the appropriate driver.
  2. Connection: Interface holding the physical connection to the DB.
  3. Statement: Interface used to execute static SQL statements.
  4. PreparedStatement: Pre-compiled statement. Highly recommended to prevent SQL injection.
  5. ResultSet: Object containing the data produced by the executed query.

2. Setup & Connecting

Ensure you have the correct connector driver `.jar` file for your database (e.g., `mysql-connector-java.jar`) in your classpath.

Java 
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class JDBCConnect {
    public static void main(String[] args) {
        String url = "jdbc:mysql://localhost:3306/my_database";
        String username = "root";
        String password = "password123";
        
        // Try-with-resources auto-closes the connection
        try (Connection conn = DriverManager.getConnection(url, username, password)) {
            System.out.println("Database connected successfully!");
            
        } catch (SQLException e) {
            System.err.println("Cannot connect to database!");
            e.printStackTrace();
        }
    }
}

3. PreparedStatement CRUD

Always use PreparedStatement over regular Statement when user input is involved to prevent SQL Injection Attacks! It also caches execution plans for better performance.
Java 
import java.sql.*;

public class JDBCCrud {
    public static void insertUser(Connection conn, String name, String email) throws SQLException {
        String sql = "INSERT INTO users (name, email) VALUES (?, ?)";
        
        try (PreparedStatement preparedStatement = conn.prepareStatement(sql)) {
            // Set the parameters (1-indexed)
            preparedStatement.setString(1, name);
            preparedStatement.setString(2, email);
            
            // Execute returns number of rows affected
            int rowsAffected = preparedStatement.executeUpdate();
            System.out.println(rowsAffected + " row(s) inserted.");
        }
    }
}

4. Result Set Handling

Java 
public static void printAllUsers(Connection conn) throws SQLException {
    String sql = "SELECT id, name, email FROM users";
    
    try (Statement statement = conn.createStatement();
         ResultSet resultSet = statement.executeQuery(sql)) {
         
        // .next() moves the cursor forward and returns true if row exists
        while (resultSet.next()) {
            int id = resultSet.getInt("id");
            String name = resultSet.getString("name");
            String email = resultSet.getString("email");
            
            System.out.println("ID: " + id + ", Name: " + name + ", Email: " + email);
        }
    }
}

5. Transaction Management

By default, JDBC auto-commits every query immediately. To bundle multiple queries (like bank transfers) so either they all succeed or all fail, manually handle commits and rollbacks.

Java 
public static void transferFunds(Connection conn, int amt, int p1, int p2) {
    try {
        // 1. Disable auto-commit
        conn.setAutoCommit(false);
        
        // 2. Execute multiple statements...
        updateAccount(conn, p1, -amt); // Deduct from User 1
        updateAccount(conn, p2, amt);  // Add to User 2
        
        // 3. Commit the transaction
        conn.commit();
        System.out.println("Transfer successful.");
        
    } catch (SQLException e) {
        try {
            // 4. If error occurs, rollback everything!
            conn.rollback();
            System.out.println("Transfer failed. Changes rolled back.");
        } catch (SQLException rollbackEx) {
            rollbackEx.printStackTrace();
        }
    } finally {
        // Always reset auto-commit
        try { conn.setAutoCommit(true); } catch (Exception ignore) {}
    }
}
